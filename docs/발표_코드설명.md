# 발표용 코드 설명서

이 문서는 최종 발표에서 "무슨 기능을 넣었는지", "어떤 함수로 만들었는지", "어떤 JavaScript 문법을 썼는지"를 빠르게 설명하기 위한 참고 자료입니다.

---

## 1. 이 프로젝트에서 추가한 핵심 기능

### 1) 공통 헤더 / 푸터 주입
- 모든 HTML에 같은 상단 메뉴와 하단 푸터를 반복해서 적지 않도록 만들었습니다.
- `assets/js/app.js`에서 `HEADER_HTML`, `FOOTER_HTML` 문자열을 한 번 정의한 뒤 각 페이지에 주입합니다.
- 덕분에 메뉴 수정이나 로고 교체를 한 곳에서만 하면 전체 페이지에 반영됩니다.

### 2) 현재 페이지 메뉴 강조
- 주소를 읽어서 현재 페이지가 무엇인지 판단한 뒤, 해당 메뉴만 `active` 상태로 바꿉니다.
- 홈, About, Products, Icheon, Archive, Stores가 페이지마다 정확히 구분되도록 했습니다.

### 3) 홈 Feature 섹션 인터랙션
- 홈의 4개 기능 카드 중 하나를 누르면 오른쪽 정보 영역이 바뀌는 구조입니다.
- 이미지, 제목, 링크, 설명이 함께 바뀌도록 구성했습니다.

### 4) About 타임라인과 Symbol 가로 스크롤
- About은 10개 타임라인으로 구성했고, Symbol은 가로로 넘기는 이미지 레일로 만들었습니다.
- 스크롤 위치에 따라 가운데에 가장 가까운 슬라이드가 강조되도록 처리했습니다.

### 5) Icheon 3페이지 순환 구조
- `icheon.html`(맛집), `icheon-discover.html`(놀거리), `icheon-festival.html`(축제)를 서로 연결했습니다.
- 상단 화살표를 누르면 다음 페이지로 넘어가고, 마지막 페이지에서 다시 처음으로 순환합니다.

### 6) 제품 탭 / 상세 페이지 / 장바구니 / 주문서
- Products는 Apparels / ACC / Icheon Rice 3개 탭으로 나뉩니다.
- 상품 상세에서는 색상과 수량을 고를 수 있고, 장바구니와 주문서로 이어집니다.
- 장바구니와 주문서는 금액 합계가 자동으로 다시 계산되도록 만들었습니다.

### 7) 로그인 / 회원가입 / 마이페이지
- 가입한 사용자 정보와 로그인 상태를 `localStorage`에 저장합니다.
- 로그인하면 상단 버튼이 `가입하기 / 로그인`에서 `마이페이지 / 로그아웃`으로 바뀝니다.

### 8) Archive 필터와 AOS 애니메이션
- News / Gallery를 필터로 나누고, 필요한 카드만 보여주도록 했습니다.
- `AOS`와 `IntersectionObserver`로 스크롤 등장 애니메이션을 적용했습니다.

## 2. 실제로 사용한 함수들

### `assets/js/app.js`의 공통 함수

#### `getPageKey()`
- 현재 주소에서 파일명을 읽어 페이지 키를 구합니다.
- 예: `index.html` → `index`, `products.html` → `products`

#### `injectLayout()`
- 헤더와 푸터를 공통 HTML로 넣는 함수입니다.
- `dataset.layoutInjected`를 사용해서 중복 주입을 막습니다.

#### `updateNavigationState(pageKey)`
- 현재 페이지에 맞는 메뉴를 활성화합니다.
- `classList.add()`와 `classList.toggle()`을 사용합니다.

#### `initFeatureTabs()`
- 홈 feature 카드 클릭 이벤트를 연결합니다.
- 카드의 `dataset` 값을 읽어서 오른쪽 내용을 바꿉니다.

#### `stageFeatureSwap(featureStage)`
- feature 영역이 바뀔 때 애니메이션을 다시 재생합니다.
- `offsetWidth`를 한 번 읽어서 강제로 리플로우를 발생시킵니다.

#### `initSymbolRail()`
- Symbol 페이지의 가로 스크롤 레일을 제어합니다.
- 휠 방향에 따라 좌우 이동을 처리하고, 중앙 슬라이드를 강조합니다.

#### `initTabs()`
- 탭 버튼과 패널을 연결합니다.
- `data-tab-target`, `data-tab-panel`을 기준으로 활성화 상태를 바꿉니다.

#### `initArchiveFilters()`
- News / Gallery 카드 필터를 제어합니다.
- 버튼에 맞는 카드만 보이도록 `hidden` 속성을 조절합니다.

#### `initAosReveal()`
- 스크롤 시 나타나는 요소를 처리합니다.
- 브라우저가 `IntersectionObserver`를 지원하지 않으면 바로 보여줍니다.

#### `getUsers()`, `setUsers()`
- 회원 목록을 `localStorage`에서 읽고 저장합니다.

#### `getAuth()`, `setAuth()`
- 현재 로그인 상태를 저장하고 불러옵니다.

#### `updateAuthNav()`
- 로그인 상태에 따라 상단 버튼 문구와 링크를 바꿉니다.
- 로그아웃 버튼에는 `data-logout` 속성을 붙여 이벤트를 연결합니다.

#### `wireAuthForms()`
- 가입 / 로그인 폼의 제출 버튼에 이벤트를 연결합니다.

#### `handleSignup(nameInput, email, password)`
- 회원가입 입력값을 검증하고 중복 이메일을 막습니다.
- 가입 성공 시 `mypage.html`로 이동합니다.

#### `handleLogin(email, password)`
- 저장된 사용자와 입력값을 비교해 로그인 처리합니다.

## 3. 페이지별로 어떤 문법을 썼는지

### Home
- `querySelector`, `querySelectorAll`
- `addEventListener('click', ...)`
- `dataset`
- `textContent`
- `setAttribute`

### About
- `classList.add`, `classList.remove`, `classList.toggle`
- `requestAnimationFrame`
- `offsetWidth`

### Products
- 탭 구조는 `data-tab-target` / `data-tab-panel`
- 상품 이미지와 버튼은 반복 구조로 정리
- 색상 / 가격 / 수량은 상세 페이지에서 상태 변화

### Product Detail
- `button` 클릭으로 색상과 이미지를 바꿈
- `toLocaleString('ko-KR')`로 가격 표시 형식을 통일
- `mainImage.src = ...`처럼 DOM 속성을 직접 바꿔서 화면을 갱신

### Cart / Checkout
- 상품 합계 계산은 `dataset.price`와 `dataset.qty`를 곱해서 계산
- `forEach()`로 각 아이템을 순회
- `innerHTML`, `textContent`를 이용해 화면 숫자를 갱신

### Login / Signup
- `localStorage.getItem()`
- `localStorage.setItem()`
- `JSON.parse()`
- `JSON.stringify()`

### Archive
- `hidden = true / false`
- 카드 필터링은 버튼 상태와 카드 속성을 비교해서 처리

### Festival / Icheon
- 상단 화살표는 일반 링크와 버튼 이벤트를 함께 사용
- 페이지 이동은 `location.href`로 처리
- 섹션 위치 맞춤은 `scroll-margin-top`으로 보정

## 4. 코드 작성 방식

### 1) 공통 구조를 먼저 만들고, 페이지별 기능은 따로 분리
- 공통 헤더, 공통 푸터, 공통 토큰은 한 번만 정의했습니다.
- 페이지별 특수 기능은 각 HTML의 하단 script에서 따로 처리했습니다.

### 2) `const` 우선, `let`은 꼭 필요할 때만
- 값이 바뀌지 않으면 `const`를 사용했습니다.
- 반복 계산이나 상태 변화가 필요한 부분만 `let`을 썼습니다.

### 3) 조건문은 중첩보다 early return
- 필요한 요소가 없으면 바로 `return`해서 코드 깊이를 줄였습니다.
- 예:

```js
const featureList = document.querySelector(".feature-list");
const featureStage = document.querySelector(".feature-stage");
if (!featureList || !featureStage) return;
```

### 4) `dataset`으로 HTML과 JS를 연결
- HTML에 `data-*` 속성을 넣고, JS에서 읽어 쓰는 방식입니다.
- 텍스트를 JS에 박아 넣기보다 구조와 데이터를 분리할 수 있습니다.

### 5) 클래스 토글로 상태 관리
- `is-active`, `header-transparent`, `header-scrolled` 같은 상태 클래스를 사용했습니다.
- 이 방식은 JavaScript가 "상태를 바꾸고", CSS가 "보이는 모양을 담당"하게 해줍니다.

### 6) 반복되는 값은 상수로 분리
- 페이지별 메뉴 매핑, 사용자 저장 키, 헤더/푸터 HTML은 상수로 빼서 재사용했습니다.
