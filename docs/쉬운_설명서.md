# 🌟 아주 쉽게 읽는 우리 웹사이트 설명서
### "비비안 × 이천쌀" 사이트를 초등학생도 이해할 수 있게!

---

## 1. 이 사이트는 무슨 사이트예요?
유명한 영국 패션 브랜드 **비비안 웨스트우드**와, 우리나라 **이천의 임금님표 쌀**이
서로 **친구가 되어 함께 만든 가짜(상상) 브랜드** 홈페이지예요. 🌾👗

> 이야기: "우주 저 멀리 '비비안 행성'에서 쌀 한 톨이 날아와, 지구의 작은 마을 **이천**에 떨어졌어요!"

---

## 2. 어떻게 만들었어요? (레고에 비유하면)
- 인터넷에는 **"이미 만들어진 레고 세트"** 같은 게 있어요. 그걸 **템플릿(Moderna)** 이라고 해요.
- 우리는 그 레고를 **부수지 않고**, 위에 **새 스티커(우리 디자인)** 를 붙였어요.
  - 원래 색(파랑) → 우리 색(**검정·하양 + 빨간 체크무늬**)
  - 원래 글씨체 → 우리 글씨체(**프리텐다드**)
- 그래서 **원래 레고 설명서(원본 파일)는 그대로** 두고, **우리 스티커 종이(custom.css, app.js)** 만 따로 만들었어요.
  - 이렇게 하면 나중에 **"원래대로 되돌리기"도 쉽고**, **어디를 바꿨는지도 한눈에** 보여요.

---

## 3. 사이트 안에는 무슨 페이지가 있어요?
- 🏠 **홈**: 첫 화면. 멋진 사진 + 영상 + 누르면 사진이 바뀌는 4칸 메뉴
- 📖 **About**: 콜라보 이야기 + 그림 타임라인 + 빙글빙글 로고
- 🛍️ **Products**: 옷·목걸이·쌀을 파는 곳 (가격이랑 "Get now" 버튼)
- 🍚 **Icheon**: 이천 맛집 / 놀거리 / 축제 (화살표 누르면 다음 페이지로 계속 넘어가요)
- 📰 **Archive**: 뉴스랑 사진 모음, 그리고 인터뷰 기사
- 📍 **Stores**: 진짜 지도로 매장 위치 보기
- 👤 **로그인·장바구니**: 가입하고, 담고, 주문하는 흉내내기

---

## 4. 클릭하면 움직이는 것들 (마법 버튼들)
- 홈의 4칸 메뉴를 누르면 → **왼쪽 큰 사진이 바뀌어요.** 사진에 마우스를 올리면 **"보러가기" 버튼**이 떠요.
- 이천 페이지의 **화살표(◀ ▶)** 를 누르면 → 맛집 → 놀거리 → 축제 → 다시 맛집… **끝없이 빙글빙글** 넘어가요.
- 가입하기를 누르면 → **내 계정이 만들어지고**, 로그인하면 위쪽 메뉴가 **"마이페이지"로 바뀌어요.**
- 갤러리 사진을 누르면 → **크게 확대**돼서 보여요.

---

## 5. 왜 이렇게 만들었어요? (똑똑한 이유)
1. **원본을 안 건드려서** 고장 나도 금방 고칠 수 있어요.
2. **색·글씨를 "변수"로** 정해놔서, 한 군데만 바꾸면 사이트 전체가 같이 바뀌어요.
3. **모든 페이지의 위(메뉴)와 아래(푸터)를 똑같이** 만들어서, 어디를 가도 똑같이 생겼어요.
4. **사진·글씨 경로를 상대경로**로 해서, 폴더째 옮겨도 안 깨져요.

---

## 6. 직접 실행해 보는 법
1. 폴더를 열어요.
2. **`index.html`** 파일을 더블클릭하면 홈 화면이 떠요!
   - (더 정확히 보려면: 터미널에서 `python3 -m http.server 5173` 입력 → 인터넷 주소창에 `http://localhost:5173/`)

## 7. 발표할 때 설명하면 좋은 "코드 구현 방식"
이 부분은 발표에서 **"이 기능은 어떤 문법으로 만들었나요?"** 라는 질문이 들어왔을 때 쓰면 좋아요.

- **공통 헤더/푸터**
  - 각 HTML에 같은 메뉴를 반복해서 쓰지 않고, `assets/js/app.js`에서 `HEADER_HTML`, `FOOTER_HTML` 문자열로 한 번 만들어 `innerHTML`로 넣었어요.
  - 핵심 문법: `document.getElementById()`, `element.innerHTML`, `if`, `dataset`
  - 이렇게 하면 모든 페이지의 위아래 구성이 같아지고, 수정도 한 곳에서 할 수 있어요.

- **메뉴가 현재 페이지를 표시하는 기능**
  - 주소를 읽어서 현재 파일 이름을 알아낸 뒤, 해당 메뉴에 `active` 클래스를 붙였어요.
  - 핵심 문법: `location.pathname`, `split()`, `replace()`, `classList.add()`, `querySelector()`
  - 예: 홈에서는 Home 메뉴가, About 페이지에서는 About 메뉴가 강조돼요.

- **홈 4칸 메뉴**
  - 오른쪽 리스트를 누르면 왼쪽 큰 이미지와 제목이 바뀌게 만들었어요.
  - 핵심 문법: `dataset`, `addEventListener('click', ...)`, `textContent`, `setAttribute()`
  - 마우스나 키보드로도 움직이게 해서 접근성을 같이 챙겼어요.

- **이천 페이지 화살표 이동**
  - 맛집, 놀거리, 축제가 서로 이어지도록 페이지 링크를 연결했어요.
  - 핵심 문법: `<a href="...">`, `onclick` 대신 일반 링크 사용
  - 그래서 화살표를 누르면 다음 섹션 페이지로 자연스럽게 넘어가요.

- **로그인/가입**
  - 입력한 값을 `localStorage`에 저장해서 새로고침해도 로그인 상태가 유지되게 했어요.
  - 핵심 문법: `localStorage.getItem()`, `localStorage.setItem()`, `JSON.parse()`, `JSON.stringify()`
  - 로그인하면 상단 메뉴가 `가입하기/로그인`에서 `마이페이지/로그아웃`으로 바뀌어요.

- **호버 효과와 애니메이션**
  - CSS의 `:hover`로 버튼이나 카드에 마우스를 올렸을 때 배경과 글씨가 바뀌게 했어요.
  - 스크롤 애니메이션은 `AOS`와 `IntersectionObserver`를 같이 사용해서, 이미지가 늦게 떠도 화면에 보이게 했어요.
  - 핵심 문법: `:hover`, `transition`, `IntersectionObserver`, `classList.add()`

- **타임라인 / 카드 / 탭**
  - 반복되는 화면은 비슷한 구조의 `div`를 여러 개 만들고, CSS Grid와 Flexbox로 정렬했어요.
  - 핵심 문법: `display: grid`, `display: flex`, `gap`, `justify-content`, `align-items`
  - 같은 규칙을 쓰면 페이지가 많아져도 모양이 흐트러지지 않아요.

- **발표에서 한 줄로 말하면**
  - "HTML은 구조를 만들고, CSS는 모양을 만들고, JavaScript는 움직임을 넣어서 구현했습니다."
